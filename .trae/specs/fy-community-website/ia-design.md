# 肥鸭战队(FY)官网社区 - 信息架构与交互设计文档

## 1. 设计目标与用户场景

### 1.1 设计目标
- **品牌传达**：通过视觉设计和内容架构，传递肥鸭战队"专业电竞+活泼亲和"的品牌调性
- **信息可达**：确保用户能在3次点击内找到任何核心信息
- **互动促进**：降低社区互动门槛，提升用户参与度和留存率
- **赛事聚焦**：强化赛事信息的展示权重，满足核心粉丝需求

### 1.2 目标用户画像

| 用户类型 | 特征 | 核心需求 | 使用频率 |
|---------|------|---------|---------|
| 新访客 | 首次了解战队，通过搜索/推荐进入 | 快速了解战队基本信息、成员构成 | 低 |
| 核心粉丝 | 关注赛事动态，追踪队员表现 | 赛程、战绩、队员数据、最新动态 | 高 |
| 社区用户 | 活跃于论坛，参与讨论互动 | 发帖回帖、互动交流、消息提醒 | 中高 |
| 管理员/运营 | 内容发布者、社区管理者 | 内容发布、数据监控、用户管理 | 中 |

### 1.3 竞品设计借鉴

| 竞品 | 可借鉴特点 | 应用方向 |
|------|-----------|---------|
| Team Liquid | 清晰的层级导航、模块化信息展示、赛事数据结构化 | 导航架构、信息卡片布局 |
| G2 Esports | 大胆的色彩运用、动态视觉元素、强烈的品牌个性 | 视觉冲击力、动效设计、品牌氛围 |
| LPL官网 | 赛事预告 prominence、实时数据更新、战队动态时间线 | 赛事模块优先级、动态信息流 |

```
肥鸭战队(FY)官网社区

│   └── 动态信息流 + 赛事预告 + 快捷入口
│
├── 战队 (Team)
│   ├── 战队介绍 (Overview)
│   ├── 成员展示 (Roster)
│   │   ├── 现役队员
│   │   ├── 教练团队
│   │   └── 荣誉墙
│   └── 战队历史 (History)
│
├── 赛事 (Matches)
│   ├── 赛程预告 (Schedule)
│   ├── 比赛结果 (Results)
│   ├── 积分榜 (Standings)
│   └── 赛事新闻 (News)
│
├── 社区 (Community)
│   ├── 论坛首页 (Forum)
│   ├── 热门话题 (Hot Topics)
│   ├── 版块分类 (Categories)
│   │   ├── 赛事讨论
│   │   ├── 战队闲聊
│   │   ├── 攻略分享
│   │   └── 创意作品

    ├── 个人资料 (Profile)

    ├── 账号设置 (Settings)
```

### 2.2 导航菜单设计

#### 顶部主导航（Primary Navigation）

```
[LOGO]  首页  战队▼  赛事▼  社区▼  [搜索图标]  [消息图标]  [用户头像▼]
```

| 菜单项 | 类型 | 下拉内容 | 说明 |
|-------|------|---------|------|
| 首页 | 链接 | - | 返回首页 |
| 战队 | 下拉菜单 | 战队介绍 / 成员展示 / 战队历史 | 展示战队核心信息 |
| 赛事 | 下拉菜单 | 赛程预告 / 比赛结果 / 积分榜 / 赛事新闻 | 赛事相关聚合入口 |
| 社区 | 下拉菜单 | 论坛首页 / 热门话题 / 版块分类 | 社区互动入口 |
| 搜索 | 图标按钮 | 展开搜索框 | 全局搜索功能 |

| 用户 | 头像下拉 | 个人资料 / 消息 / 收藏 / 设置 / 退出 | 个人中心入口 |

#### 移动端导航（Mobile Navigation）

- 汉堡菜单图标展开全屏抽屉式导航
- 二级菜单以手风琴折叠面板呈现
- 底部固定快捷栏：首页 / 赛事 / 社区 / 我的

### 2.3 面包屑导航（Breadcrumb）

适用于深层级页面，位于页面标题上方：

```
首页 > 社区 > 版块分类 > 赛事讨论
```

## 3. 核心用户流程

### 3.1 新用户浏览流程（发现路径）

```
[入口：搜索引擎/社交媒体/直接访问]
         │
         ▼
    [首页 Landing]
    ├─ 战队简介横幅（快速了解）
    ├─ 下一场赛事预告（倒计时）
    ├─ 最新动态/新闻
    └─ 快捷入口卡片
         │
    ┌────┴────┐
    ▼         ▼
[战队介绍]  [赛事信息]
    │         │
    ▼         ▼
[成员展示]  [赛程详情]
    │         │
    ▼         ▼
[队员详情]  [比赛数据]
    │         │
    └────┬────┘
         ▼
    [注册/关注]
    └─ 引导注册会员
    └─ 关注赛事提醒
```

**设计要点**：
- 首页首屏必须包含战队核心信息和赛事预告
- 每个内容区块提供清晰的"了解更多"入口
- 在浏览路径中适时插入注册引导（非强制弹窗，采用内嵌推荐形式）

### 3.2 老用户互动流程（参与路径）

```
[入口：直接访问/书签/推送通知]
         │
         ▼
    [首页/社区/赛事]
         │
    ┌────┴────┐
    ▼         ▼
[社区论坛]  [赛事页面]
    │         │
    ▼         ▼
[浏览帖子]  [查看战报]
    │         │
    ▼         ▼
[发表评论]  [参与讨论]
    │         │
    ▼         ▼
[互动反馈]  [分享转发]
    │         │
    └────┬────┘
         ▼
    [消息中心]
    └─ 查看回复提醒
    └─ 查看点赞通知
    └─ 管理互动消息
```

**设计要点**：
- 社区帖子支持快速回复（无需跳转新页面）
- 消息通知实时推送，提供一键已读功能
- 互动操作（点赞/回复）提供即时视觉反馈

### 3.3 管理员发布流程（内容管理路径）

```
[入口：管理员登录]
         │
         ▼
    [管理后台/前端发布入口]
         │
    ┌────┼────┐
    ▼    ▼    ▼
[发布新闻] [发布赛程] [管理帖子]
    │        │        │
    ▼        ▼        ▼
[编辑内容] [填写信息] [审核内容]
    │        │        │
    ▼        ▼        ▼
[设置标签] [选择队伍] [处理举报]
    │        │        │
    ▼        ▼        ▼
[预览发布] [确认发布] [执行操作]
    │        │        │
    └────────┴────────┘
             │
             ▼
        [内容上线]
        └─ 推送通知给关注用户
        └─ 更新相关页面数据
```

**设计要点**：
- 提供前端快捷发布入口（富文本编辑器）
- 支持内容预览和定时发布
- 发布后自动触发消息推送

## 5. 交互状态说明

### 5.1 全局交互状态

#### 导航交互

| 元素 | 状态 | 视觉表现 | 触发方式 |
|------|------|---------|---------|
| 导航链接 | 默认 | 文字颜色 #B8860B（棕黄） | - |
| 导航链接 | 悬停 | 文字颜色 #FFD700（金黄）+ 下划线动画 | mouseenter |
| 导航链接 | 激活/当前页 | 文字颜色 #FFD700 + 底部2px指示条 | 当前页面 |
| 下拉菜单 | 展开 | 下拉面板淡入 + 轻微下移动画 | click/hover |
| 下拉菜单 | 收起 | 面板淡出 + 上移 | click outside/leave |

#### 按钮交互

| 按钮类型 | 状态 | 视觉表现 | 说明 |
|---------|------|---------|------|
| 主按钮（金黄） | 默认 | 背景 #FFD700，文字深色 | 主要CTA |
| 主按钮 | 悬停 | 背景 #FFEB3B，轻微上浮阴影 | 增强点击欲望 |
| 主按钮 | 按下 | 背景 #FF8F00，阴影内收 | 按下反馈 |
| 主按钮 | 禁用 | 背景 #FFF8E1，文字灰色 | 不可点击 |
| 次按钮（线框） | 默认 | 边框 #B8860B，透明背景 | 次要操作 |
| 次按钮 | 悬停 | 背景 #FFF8E1，边框 #FFD700 | 提示可点击 |
| 幽灵按钮 | 默认 | 透明背景，白色文字 | 深色背景上使用 |
| 幽灵按钮 | 悬停 | 背景 rgba(255,255,255,0.1) | 轻微高亮 |

#### 链接交互

| 状态 | 视觉表现 |
|------|---------|
| 默认 | 颜色 #B8860B，无下划线 |
| 悬停 | 颜色 #FFD700，显示下划线，0.2s过渡 |
| 已访问 | 颜色 #8B6914（更深） |
| 禁用 | 颜色 #CCC，cursor: not-allowed |

### 5.2 组件级交互状态

#### 卡片交互（赛事卡片/成员卡片/帖子卡片）

| 状态 | 视觉表现 | 动画参数 |
|------|---------|---------|
| 默认 | 正常位置，标准阴影 | - |
| 悬停 | translateY(-4px)，阴影增强 | 0.2s ease-out |
| 按下 | translateY(-2px)，阴影减弱 | 0.1s |
| 加载中 | 骨架屏脉冲动画 | 1.5s infinite |
| 错误 | 边框变红，显示重试按钮 | - |

#### 表单输入框交互

| 状态 | 视觉表现 |
|------|---------|
| 默认 | 边框 1px #DDD，背景白色 |
| 聚焦 | 边框 2px #FFD700，外发光阴影 |
| 有效 | 边框 #4CAF50，右侧显示对勾图标 |
| 无效 | 边框 #F44336，下方显示错误提示 |
| 禁用 | 背景 #F5F5F5，文字灰色 |
| 加载 | 输入框半透明，显示加载图标 |

#### 标签/徽章交互

| 状态 | 视觉表现 |
|------|---------|
| 默认 | 背景色 + 圆角 |
| 悬停 | 背景色加深，cursor: pointer |
| 选中 | 边框高亮，背景色饱和 |
| 可关闭 | 右侧显示 × 图标，悬停变红 |

### 5.3 页面级交互状态

#### 加载状态

| 场景 | 加载方式 | 说明 |
|------|---------|------|
| 页面初始加载 | 全屏Loading动画（战队Logo+进度条） | 首次访问 |
| 页面切换 | 顶部进度条（NProgress风格） | 路由切换 |
| 数据加载 | 骨架屏（Skeleton Screen） | 内容区域 |
| 图片加载 | 模糊占位图 -> 清晰图过渡 | 渐进加载 |
| 无限滚动 | 底部加载更多指示器（旋转动画） | 列表加载 |
| 提交操作 | 按钮Loading状态（文字变旋转图标） | 表单提交 |

#### 空状态

| 场景 | 展示内容 |
|------|---------|
| 无搜索结果 | 吉祥物插图 + "没有找到相关内容" + 搜索建议 |
| 无通知 | 铃铛图标 + "暂无消息" + 引导文字 |
| 无帖子 | 文档图标 + "还没有帖子" + 发布引导按钮 |
| 网络错误 | 警告图标 + "网络连接失败" + 重试按钮 |
| 404页面 | 吉祥物困惑表情 + "页面走丢了" + 返回首页 |

#### 反馈状态

| 操作 | 反馈方式 | 持续时间 |
|------|---------|---------|
| 点赞成功 | 心形图标放大动画 + 数字+1动画 | 0.5s |
| 收藏成功 | 星星图标旋转填充 | 0.3s |
| 评论发布 | 新评论滑入列表顶部 + 成功提示 | 3s提示 |
| 复制链接 | Toast提示"链接已复制" | 2s |
| 关注战队 | 按钮状态切换 + 成功Toast | 2s |
| 表单提交成功 | Toast + 页面跳转/刷新 | 2s |
| 表单提交失败 | 错误提示 + 抖动动画 | 持续显示 |

### 5.4 动画与过渡规范

#### 过渡时间标准

| 类型 | 时长 | 用途 |
|------|------|------|
| 微交互（按钮、图标） | 150-200ms | 悬停、点击反馈 |
| 组件状态切换 | 200-300ms | 展开/收起、显示/隐藏 |
| 页面过渡 | 300-400ms | 路由切换、模态框 |
| 复杂动画 | 500-800ms | 数据可视化、引导动画 |

#### 缓动函数标准

| 名称 | 值 | 用途 |
|------|-----|------|
| 标准 | ease-in-out | 大多数过渡 |
| 弹出 | cubic-bezier(0.175, 0.885, 0.32, 1.275) | 按钮点击、徽章 |
| 平滑 | cubic-bezier(0.4, 0, 0.2, 1) | 页面切换、模态框 |
| 减速 | cubic-bezier(0, 0, 0.2, 1) | 元素进入 |
| 加速 | cubic-bezier(0.4, 0, 1, 1) | 元素离开 |

#### 滚动行为

| 场景 | 行为 |
|------|------|
| 页面内锚点跳转 | 平滑滚动（smooth scroll） |
| 返回顶部 | 固定右下角按钮，点击平滑滚动到顶 |
| 无限滚动加载 | 接近底部时自动加载，显示加载指示器 |
| 横向滚动区域 | 鼠标滚轮映射为横向滚动，支持拖拽 |

## 7. 无障碍设计（Accessibility）

### 7.1 色彩对比度

- 正文文字与背景对比度 >= 4.5:1
- 大文字/标题对比度 >= 3:1
- 交互元素标识不仅依赖颜色（图标+文字）

### 7.2 键盘导航

- 所有交互元素支持Tab键聚焦
- 聚焦状态有清晰的视觉指示（outline或阴影）
- 下拉菜单支持方向键导航
- 模态框支持ESC键关闭

### 7.3 屏幕阅读器

- 图片提供有意义的alt文本
- 图标按钮提供aria-label
- 动态内容更新使用aria-live通知
- 表单输入关联label或使用aria-labelledby

### 7.4 动画偏好

- 支持 prefers-reduced-motion 媒体查询
- 减少动画模式下，所有过渡变为瞬时切换
- 不将关键信息仅通过动画传达
