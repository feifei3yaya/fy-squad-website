# 肥鸭战队(FY)官网社区 - 信息架构与交互设计文档

## 1. 设计目标

- **品牌传达**：通过视觉设计和内容架构传递"专业电竞+活泼亲和"的品牌调性
- **信息可达**：确保用户能在3次点击内找到任何核心信息
- **互动促进**：降低社区互动门槛，提升用户参与度
- **赛事聚焦**：强化赛事信息的展示权重

## 2. 网站层级结构

```
肥鸭战队(FY)官网社区
├── 首页 (Home) - 动态信息流 + 赛事预告 + 快捷入口
├── 战队 (Team) - 战队介绍 / 成员展示 / 战队历史
├── 赛事 (Matches) - 赛程预告 / 比赛结果 / 积分榜 / 赛事新闻
├── 社区 (Community) - 论坛首页 / 热门话题 / 版块分类
├── 商城 (Store) [可选扩展]
└── 用户中心 (User) - 个人资料 / 消息通知 / 收藏 / 设置
```

### 导航菜单设计
```
[LOGO]  首页  战队▼  赛事▼  社区▼  [搜索]  [消息]  [用户头像▼]
```

### 移动端导航
- 汉堡菜单展开全屏抽屉式导航
- 二级菜单手风琴折叠面板
- 底部固定快捷栏

## 3. 核心用户流程

### 3.1 新用户浏览流程
搜索引擎/社交媒体 → 首页 Landing → 战队介绍 → 成员展示 → 队员详情 → 注册/关注

### 3.2 老用户互动流程
首页/社区 → 浏览帖子 → 发表评论 → 互动反馈 → 分享转发 → 消息中心

### 3.3 管理员发布流程
管理员登录 → 管理后台 → 发布新闻/赛程/管理帖子 → 预览发布 → 内容上线

## 4. 主要页面布局

### 4.1 首页（Home）- P0 优先级
- Hero Banner（战队主题视觉区 + 倒计时 + CTA按钮）
- 赛事预告区（横向滚动卡片）
- 战队动态/新闻区（双栏布局）
- 成员展示区（网格）
- 社区热门话题区
- 战队荣誉/数据区

### 4.2 战队介绍页（Team Overview）- P1
- 战队视觉Banner + Logo
- 价值观/文化区（三栏图标卡片）
- 历史时间线（垂直时间轴）
- 核心成就区（荣誉墙）

### 4.3 成员展示页（Team Roster）- P1
- 筛选标签栏（按位置筛选）
- 成员卡片网格（PC:4列 / 平板:2列 / 手机:1列）
- 卡片含头像、ID、位置、擅长英雄、数据

### 4.4 赛事信息页（Matches）- P1
- Tab切换：赛程预告 | 比赛结果 | 积分榜 | 新闻
- 视图切换：日历视图 | 列表视图
- 赛事卡片含双方Logo、比分、状态

### 4.5 社区论坛页（Forum）- P1
- 左侧固定边栏（发布按钮 + 版块导航 + 热门标签）
- 主内容区（筛选栏 + 帖子列表 + 无限滚动）

### 4.6 帖子详情页（Post Detail）- P1
- 面包屑导航 → 帖子头部 → 正文 → 互动操作栏 → 评论区

### 4.7 用户中心页（Profile）- P2
- 顶部信息卡片 + Tab内容区

### 4.8 消息通知中心（Notifications）- P2
- 筛选标签 + 通知列表

## 5. 交互状态说明

### 全局交互状态
- 导航链接：默认/悬停/激活
- 按钮：默认/悬停/按下/禁用
- 链接：默认/悬停/已访问/禁用

### 组件级交互
- 卡片：默认/悬停(上浮4px)/按下/加载/错误
- 表单输入框：默认/聚焦/有效/无效/禁用/加载
- 标签/徽章：默认/悬停/选中/可关闭

### 页面级交互
- 加载状态：全屏Loading、顶部进度条、骨架屏、图片渐进加载
- 空状态：各种场景的吉祥物插图 + 引导文字
- 反馈状态：点赞动画、Toast提示、表单错误抖动

## 6. 响应式断点

| Mobile (<640px) | Tablet (640-1024px) | Desktop (1024-1440px) | Large (>1440px) |

## 7. 无障碍设计

- 色彩对比度 ≥ 4.5:1
- 键盘导航支持
- 屏幕阅读器支持
- prefers-reduced-motion 动画偏好
