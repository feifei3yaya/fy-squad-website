# 肥鸭战队（FY）官网社区视觉设计规范

## 概述

本规范定义了肥鸭战队（FY）官网社区的完整视觉系统，涵盖色彩、字体、LOGO、吉祥物、间距、阴影、图标等核心设计要素。整体风格定位为**现代电竞风 + 活泼可爱元素**，通过明快的黄色主色调传递战队的活力与亲和力，同时以深色背景营造沉浸式电竞氛围。

---

## 2. LOGO 设计规范

### 2.1 LOGO 概念描述

肥鸭战队LOGO由**肥鸭形象**与 **FY 字母组合**构成，整体呈现一只圆润可爱的黄色小鸭侧身站立姿态，鸭身自然融入字母 F 的负形空间，鸭喙上方巧妙延伸出字母 Y 的笔画，形成图形与文字的有机融合。

**核心元素：**
- **鸭头**：圆润饱满，带有一撮翘起的呆毛，眼睛为简化的斜线战斗眼（`> <` 形状），传递电竞战斗感
- **鸭身**：肥硕圆润的轮廓，腹部形成字母 F 的横竖结构
- **鸭尾**：上翘的鸭尾羽化作为字母 Y 的分叉笔画
- **鸭掌**：稳立于基线之上，底部可搭配装饰性平台

### 2.2 LOGO 标准型

- **主色版本**：鸭身为 `#FFC107` 金黄渐变至 `#FFE14D` 亮黄，轮廓描边为 `#0A0A0F` 深黑，描边粗细 2px
- **文字组合**：图形右侧或下方搭配 "FY" 字母，使用得意黑体，字重 700，字间距 -0.02em
- **战队全称**：图形 + "FY" + "肥鸭战队" 横排组合，用于正式场合

### 2.3 LOGO 反白型

- 用于深色背景或图片背景之上
- 鸭身使用纯白 `#FFFFFF`，保留 `#0A0A0F` 深色描边以确保辨识度
- 文字同样反白处理

### 2.4 最小尺寸规范

| 应用场景 | 最小宽度 | 备注 |
|----------|----------|------|
| 网页头部 | 120px | 保持鸭眼和 FY 字母清晰可辨 |
| 页脚/次级位置 | 80px | 允许简化版本（仅图形或仅文字） |
| 社交媒体头像 | 64px | 使用图形单独版本 |
|  favicon | 32px | 提取鸭头简化图形 |

### 2.5 安全空间

- LOGO 四周需保留最小等于鸭头高度的空白区域
- 禁止在安全空间内放置其他图形、文字或界面元素
- 禁止拉伸、旋转、倾斜、添加阴影或外发光效果（规范版本除外）

### 2.6 禁止使用场景

- 不得改变品牌色的色相（如改为绿色、红色鸭）
- 不得移除描边导致图形与背景融合
- 不得单独使用鸭头而不带 FY 标识（社交媒体头像除外）
- 不得将 LOGO 作为内容填充在密集纹理之上

## 3. 吉祥物形象设计

### 3.1 基础造型

吉祥物名为**"肥肥"**（Fatty），是一只圆滚滚的黄色小鸭子，整体比例约为 2:1.8（宽:高），呈现 Q 版三头身比例。

**身体特征：**
- **体型**：极度圆润，腹部突出，几乎看不到脖子，翅膀为小短翼贴在身体两侧
- **颜色**：主体为 `#FFE14D` 亮黄，腹部略浅为 `#FFF8E1` 奶油黄，形成柔和过渡
- **鸭喙**：橙黄色 `#FF9800`，宽大扁平，微微上翘呈现微笑感
- **鸭掌**：橙黄色 `#FF9800`，短粗圆润，站立时呈外八字
- **呆毛**：头顶有一撮翘起的羽毛，为标志性特征，使用 `#FFC107`

### 3.2 面部表情系统

| 表情名称 | 特征描述 | 使用场景 |
|----------|----------|----------|
| 战斗眼 | 眼睛为 `> <` 形状，眉毛下压，鸭喙紧闭 | 比赛、竞技、对战相关 |
| 开心眼 | 眼睛为弯月形 `^ ^`，嘴巴张开大笑 | 胜利、庆祝、欢迎页面 |
| 星星眼 | 眼睛为星星形状 `★ ★`，嘴巴微张 | 获得奖励、开箱、惊喜时刻 |
| 困困眼 | 眼睛为半闭线形 `- -`，带小腮红 | 夜间模式、加载等待、休息状态 |
| 哭泣眼 | 眼睛为 `Q Q` 带泪珠，嘴巴下撇 | 失败、掉线、错误提示 |
| 得意眼 | 单眼 wink `> ^`，嘴角上扬 | 成就达成、排行榜展示 |

### 3.3 服装与道具变装

| 变装主题 | 描述 | 应用场景 |
|----------|------|----------|
| 电竞选手装 | 头戴电竞耳机，身穿战队队服（黑黄配色），手持游戏手柄 | 赛事页面、直播入口 |
| 指挥官装 | 头戴贝雷帽，手持小旗子，表情严肃 | 战队管理、战术板页面 |
| 应援装 | 手持荧光棒，头戴发箍，表情激动 | 粉丝社区、应援活动 |
| 节日限定 | 根据节日变换装饰（如春节戴虎头帽、圣诞戴鹿角） | 节日活动页面 |

### 3.4 应用场景

| 场景 | 使用形式 | 尺寸建议 |
|------|----------|----------|
| 404 错误页 | 哭泣表情肥肥 + 提示文字 | 240px 高度 |
| 加载动画 | 旋转的肥肥 + 进度条 | 80px 高度 |
| 空状态页 | 对应主题表情的肥肥 + 引导文案 | 200px 高度 |
| 成就徽章 | 肥肥头像 + 成就图标组合 | 64px 圆形 |
| 消息通知 | 小尺寸肥肥头像 + 消息气泡 | 40px 圆形 |
| 背景装饰 | 半透明肥肥剪影散布 | 120px 随机分布 |

## 4. 字体排版规范

### 4.1 字体家族

| 用途 | 字体 | 备选字体 | 说明 |
|------|------|----------|------|
| 标题字体 | 得意黑（Smiley Sans） | 思源黑体 Heavy | 几何感强，倾斜设计传递速度感与电竞气质 |
| 正文字体 | 思源黑体（Source Han Sans） | PingFang SC, Microsoft YaHei | 清晰易读，覆盖全字重，适合长文本阅读 |
| 数字/数据 | DIN Alternate / Roboto Mono | Arial | 等宽或比例数字，确保数据对齐与科技感 |
| 英文标题 | Inter / Montserrat | Arial | 现代无衬线，与得意黑气质匹配 |

### 4.2 字号层级

| 层级 | 字号 | 字重 | 行高 | 字间距 | 用途 |
|------|------|------|------|--------|------|
| H1 英雄标题 | 48px / 3rem | 700 | 1.1 | -0.02em | 首页英雄区大标题 |
| H2 板块标题 | 36px / 2.25rem | 700 | 1.2 | -0.01em | 页面主要板块标题 |
| H3 卡片标题 | 24px / 1.5rem | 600 | 1.3 | 0 | 卡片标题、模块标题 |
| H4 小标题 | 18px / 1.125rem | 600 | 1.4 | 0 | 列表标题、表单分组 |
| Body 正文 | 16px / 1rem | 400 | 1.6 | 0.01em | 正文段落、描述文字 |
| Small 辅助 | 14px / 0.875rem | 400 | 1.5 | 0.01em | 辅助说明、元信息 |
| Caption 标注 | 12px / 0.75rem | 400 | 1.4 | 0.02em | 时间戳、标签、角标 |

### 4.3 响应式字号

| 断点 | H1 | H2 | H3 | Body |
|------|-----|-----|-----|------|
| Desktop (>=1280px) | 48px | 36px | 24px | 16px |
| Tablet (>=768px) | 40px | 30px | 22px | 16px |
| Mobile (<768px) | 32px | 26px | 20px | 15px |

### 4.4 排版规则

- **标题颜色**：H1-H2 使用 `#E8E8F0`，H3-H4 可使用 `#FFE14D` 进行强调
- **正文颜色**：统一使用 `#E8E8F0`，确保深色背景可读性
- **链接样式**：默认 `#00D4FF`，悬停时添加下划线，visited 状态保持原色
- **高亮文本**：使用 `#FFE14D` 背景色块（`background: rgba(255, 225, 77, 0.15)`）或文字色
- **文字阴影**：标题可使用 `text-shadow: 0 2px 8px rgba(0,0,0,0.5)` 增强层次感

## 5. 间距与圆角规范

### 5.1 间距系统

基于 4px 基准单位，建立完整的间距尺度：

| Token | 数值 | 用途 |
|-------|------|------|
| space-1 | 4px | 图标与文字间距、紧凑内联间距 |
| space-2 | 8px | 按钮内边距（垂直）、小标签间距 |
| space-3 | 12px | 表单元素间距、小卡片内边距 |
| space-4 | 16px | 标准卡片内边距、列表项间距 |
| space-5 | 20px | 中等容器内边距 |
| space-6 | 24px | 大卡片内边距、模块间距 |
| space-8 | 32px | 板块内部间距 |
| space-10 | 40px | 板块之间间距 |
| space-12 | 48px | 大模块间距 |
| space-16 | 64px | 页面级垂直间距 |
| space-20 | 80px | 英雄区上下间距 |

### 5.2 布局网格

- **桌面端**：12 列网格，列宽 72px，间距 24px，总宽 1128px（居中）
- **平板端**：8 列网格，列宽自适应，间距 20px
- **移动端**：4 列网格，列宽自适应，间距 16px

### 5.3 圆角规范

| Token | 数值 | 用途 |
|-------|------|------|
| radius-sm | 4px | 小按钮、标签、输入框 |
| radius-md | 8px | 标准按钮、卡片、下拉菜单 |
| radius-lg | 12px | 大卡片、模态框、图片容器 |
| radius-xl | 16px | 英雄卡片、特色区块 |
| radius-full | 9999px | 胶囊按钮、头像、徽章 |

### 5.4 圆角使用原则

- 卡片与容器：统一使用 `radius-md` (8px) 或 `radius-lg` (12px)
- 按钮：主按钮使用 `radius-md`，胶囊标签使用 `radius-full`
- 图片：根据容器圆角保持一致，可使用 `overflow: hidden`
- 避免在同一界面混合使用过多不同圆角值，保持视觉统一

## 6. 阴影与渐变规范

### 6.1 阴影系统

| Token | 定义 | 用途 |
|-------|------|------|
| shadow-sm | `0 1px 2px rgba(0,0,0,0.3)` | 小元素悬浮、下拉选项 |
| shadow-md | `0 4px 12px rgba(0,0,0,0.4)` | 卡片悬浮、按钮悬浮 |
| shadow-lg | `0 8px 24px rgba(0,0,0,0.5)` | 模态框、悬浮面板 |
| shadow-xl | `0 16px 48px rgba(0,0,0,0.6)` | 全屏弹窗、重要通知 |
| shadow-glow | `0 0 20px rgba(255, 193, 7, 0.3)` | 主按钮发光、选中状态 |
| shadow-blue-glow | `0 0 16px rgba(0, 212, 255, 0.25)` | 链接悬浮、信息元素发光 |

### 6.2 阴影使用原则

- 深色背景下阴影使用纯黑 `rgba(0,0,0,?)`，确保一致性
- 发光阴影仅用于强调交互元素（主按钮、选中态）
- 避免在同一元素叠加多层阴影造成脏感
- 卡片默认无阴影，悬浮时过渡至 `shadow-md`

### 6.3 渐变详细规范

**主品牌渐变（按钮/LOGO）：**
```css
background: linear-gradient(135deg, #FFE14D 0%, #FFC107 50%, #FF9800 100%);
```

**卡片顶部光晕：**
```css
background: linear-gradient(180deg, rgba(255,225,77,0.08) 0%, transparent 100%);
```

**霓虹边框渐变：**
```css
border-image: linear-gradient(90deg, #00D4FF, #B829F7) 1;
```

**深色背景过渡：**
```css
background: linear-gradient(180deg, #0A0A0F 0%, #14141B 50%, #0A0A0F 100%);
```

### 6.4 渐变使用原则

- 品牌渐变仅用于 CTA 按钮、LOGO、核心视觉元素，避免滥用
- 背景渐变使用低饱和度、低透明度，确保内容可读性
- 霓虹渐变仅用于特殊活动、限定内容，制造视觉焦点
- 渐变方向保持统一（优先 135deg 对角或 180deg 垂直）

## 7. 图标风格规范

### 7.1 图标风格

- **风格**：线性图标（Outline Style）为主，2px 描边，端点圆角（round cap/join）
- **视觉平衡**：图标在 24x24 网格中绘制，保持视觉重心居中
- **统一性**：所有图标保持相同的描边粗细、端点样式、圆角大小
- **双色点缀**：允许在图标局部使用品牌黄色 `#FFC107` 作为点缀色（如星星、火焰、闪电元素）

### 7.2 图标尺寸

| 尺寸 | 用途 |
|------|------|
| 16px | 内联文字图标、按钮内部小图标 |
| 20px | 表单前缀图标、紧凑列表图标 |
| 24px | 标准导航图标、功能图标 |
| 32px | 大按钮图标、特色功能入口 |
| 48px | 空状态图标、成就图标 |
| 64px | 英雄区装饰图标、大功能模块 |

### 7.3 图标状态

| 状态 | 样式 |
|------|------|
| 默认 | 描边色 `#8A8A9A`，2px 粗细 |
| 悬浮 | 描边色 `#E8E8F0`，可添加 `shadow-glow` |
| 激活/选中 | 描边色 `#FFE14D`，可填充 `rgba(255,225,77,0.1)` |
| 禁用 | 描边色 `#5A5A6A`，无交互反馈 |

### 7.4 图标库建议

- 基础图标：使用 **Phosphor Icons** 或 **Heroicons**（线条风格匹配）
- 游戏/电竞专属：定制图标（奖杯、头盔、武器、地图标记等）
- 吉祥物图标：提取肥肥的简化轮廓作为特殊功能图标

### 7.5 图标与文字组合

- 图标与文字间距：8px（`space-2`）
- 图标垂直居中对齐文字基线
- 按钮内图标在前时，整体左对齐，间距 8px
- 导航项图标在上、文字在下的布局时，间距 4px

## 9. 视觉风格原则

### 9.1 现代电竞风体现

- **深色沉浸式背景**：以 `#0A0A0F` 为基底，营造专注的电竞氛围
- **高对比度文字**：月白文字在深色背景上确保清晰可读
- **霓虹光效点缀**：蓝色与紫色的发光效果传递科技感
- **几何感排版**：得意黑的倾斜设计与锐利几何感传递速度与现代感
- **数据可视化**：使用等宽数字字体，强调战绩、排名等数据的展示

### 9.2 活泼可爱元素体现

- **圆润造型语言**：大圆角卡片、胶囊按钮、圆润的图标端点
- **吉祥物渗透**：肥肥形象贯穿错误页、加载态、空状态、通知等场景
- **明快黄色基调**：品牌黄色传递乐观、活力、亲和的团队气质
- **表情系统**：丰富的面部表情让界面具有情感温度
- **微交互动效**：按钮按压弹性、卡片悬浮抬升、图标旋转等活泼动效

### 9.3 视觉层次构建

1. **第一层（背景）**：深色基底 + 微弱纹理/渐变，不抢夺注意力
2. **第二层（容器）**：暗夜灰卡片，明确内容边界
3. **第三层（内容）**：月白文字 + 银灰辅助文字，信息层级清晰
4. **第四层（强调）**：黄色品牌元素 + 蓝色链接，引导视觉焦点
5. **第五层（交互）**：发光阴影、悬浮状态，反馈用户操作
