# 项目全局规则

## 0. 核心沟通与协作规则

### 0.1 语言规范
- 与用户的所有沟通必须使用中文
- 代码注释使用中文，与用户沟通保持一致

### 0.2 自动化原则
- 全程不要让用户动手，所有能自动化完成的任务都主动执行
- 能自动化解决的问题不要让用户动手
- 优先主动完成任务，减少用户操作步骤
- 对于简单直接的需求，直接执行不需要额外确认
- 主动探索和解决问题，不要等待用户指示
- 尽可能一步到位完成任务，避免让用户多次确认
- 仅在必须用户动手的部分（如敏感操作确认、提供密钥等）才需要用户参与

## 1. 代码风格规范

### 1.1 通用原则
- 遵循项目现有的代码风格和约定
- 保持代码简洁、可读、可维护
- 优先使用项目已有的工具库和框架

### 1.2 注释规范
- 为公共 API 添加清晰的文档注释
- 复杂逻辑添加必要的行内注释
- 注释使用中文，与用户沟通保持一致

## 2. 开发工作流

### 2.1 任务规划
- 对于多步骤任务，使用 TodoWrite 工具进行规划
- 优先使用搜索工具了解现有代码库结构
- 在修改代码前先阅读相关文件

### 2.2 代码修改
- 优先使用 SearchReplace 工具编辑现有文件
- 避免不必要的文件创建
- 遵循现有代码的导入、命名和结构模式

### 2.3 测试与验证
- 代码修改后运行项目的 lint 和 typecheck 命令
- 如果没有配置 lint/typecheck，询问用户验证方式

## 3. 安全最佳实践

- 永远不要提交或暴露密钥、密码等敏感信息
- 遵循安全编码规范
- 输入验证和输出转义

## 4. Git 操作

- 除非用户明确要求，否则不执行 git commit 操作
- 如果需要提交，先询问用户确认
