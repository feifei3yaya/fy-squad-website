# 肥鸭战队官网 - 技术规范、排期与测试标准

## 1. 接口定义规范 (API Standards)
所有板块接口必须遵循统一的 RESTful 规范。

### 1.1 数据交互格式
所有接口固定返回 JSON 格式，统一包裹层：
```json
{
  "code": 200,                  // 业务状态码 (200: 成功, 401: 未授权, 403: 无权限, 500: 服务器错误)
  "message": "success",         // 提示信息
  "data": { ... },              // 核心数据载荷
  "traceId": "req-12345abcde"   // 链路追踪ID，用于排查日志
}
```

### 1.2 认证规范
- 采用 `JWT (JSON Web Token)` 在 `Authorization: Bearer <token>` 请求头中传递。
- 敏感接口需在 Header 中附加 `X-CSRF-Token` 防止跨站请求伪造。

## 2. 测试用例清单 (Test Cases Matrix)

### 2.1 安全性测试 (Security)
- [ ] **TC-SEC-01**: 越权测试 - 尝试使用普通用户 Token 访问 `DELETE /api/admin/users`，预期返回 403。
- [ ] **TC-SEC-02**: 防刷测试 - 并发发送 100 次发帖请求，预期触发 429 Too Many Requests 拦截。
- [ ] **TC-SEC-03**: XSS 防御 - 在百科词条中提交 `<script>alert(1)</script>`，预期被自动转义或拦截。
- [ ] **TC-SEC-04**: SQL 注入 - 在论坛搜索框输入 `' OR 1=1 --`，预期系统正常处理，不抛出数据库异常。

### 2.2 核心业务功能测试 (Functional)
- [ ] **TC-ADM-01**: 操作审计 - 管理员封禁用户后，检查系统审计日志是否准确记录 Action、IP 和操作人。
- [ ] **TC-WIK-01**: 版本回滚 - 编辑词条并保存，在历史记录中点击回滚上一版本，预期内容精准恢复。
- [ ] **TC-SPO-01**: 支付回调掉单处理 - 模拟微信支付回调超时，通过后台“补单”按钮输入微信单号，预期系统正确下发游戏 VIP 权益。
- [ ] **TC-JOIN-01**: 状态机测试 - 拒绝加入申请后，再次尝试将其状态改为“考核中”，预期系统拒绝不合理的状态跃迁。

### 2.3 性能与压测 (Performance)
- [ ] **TC-PERF-01**: 论坛首页在 500 QPS 并发下，接口响应时间 95 分位 < 200ms。
- [ ] **TC-PERF-02**: 服务器状态 WebSocket 推送在 2000 客户端同时连接下，CPU 占用率 < 60%。

## 3. 开发排期节点 (Gantt Schedule)

项目分为 4 个主要里程碑（Milestones），建议采用敏捷迭代方式。

### Phase 1: 基础设施与后台核心 (Week 1 - Week 2)
- 数据库表结构设计与建表。
- RBAC 权限系统开发、JWT 鉴权中间件。
- 审计日志模块与 Redis 限流中间件。
- 后台基础框架搭建与数据脱敏组件。

### Phase 2: 内容双擎 - 百科与论坛 (Week 3 - Week 4)
- 富文本/Markdown 编辑器集成。
- 词条版本控制与审核流引擎。
- 论坛跟帖、楼中楼、点赞系统。
- 第三方内容风控 API 对接。

### Phase 3: 业务闭环 - 赞助与招募 (Week 5 - Week 6)
- 微信/支付宝支付 SDK 接入与异步回调处理。
- 游戏服 RCON 接口对接（自动发放权益）。
- 招募表单、审核状态机及站内信系统。

### Phase 4: 监控面板与联调测试 (Week 7)
- 前后台 API 联调对接。
- 服务器状态监控数据采集与 WebSocket 推送。
- 全量自动化测试、渗透测试及 Bug 修复。
- 生产环境部署（按架构规范配置 WAF 与 LB）。
