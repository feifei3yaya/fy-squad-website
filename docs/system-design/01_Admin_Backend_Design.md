# 肥鸭战队官网 - 后台核心板块设计规范 (P0)

## 1. 整体架构与安全设计
后台系统是整个社区的控制中枢，采用微服务/模块化架构，严格遵循“最小权限原则”与“默认拒绝策略”。

### 1.1 登录与认证安全
- **多因素认证 (2FA)**：超管及核心操作员强制开启 TOTP (基于时间的一次性密码) 或短信/邮件验证码。
- **异常拦截机制**：
  - 连续登录失败 5 次，账号锁定 15 分钟。
  - 异地登录告警（基于 IP 地理位置库库比对）。
  - 登录会话超时机制（Token 有效期 2 小时，无操作 30 分钟自动登出）。
- **接口防刷 (Rate Limiting)**：关键写操作接口（如发帖、审核）限制 `5次/秒`，普通读接口限制 `50次/秒`（基于 Redis 令牌桶算法）。

### 1.2 数据脱敏规范
- **敏感信息存储**：用户密码必须使用 `Argon2` 或 `Bcrypt` 加密存储，绝不存储明文。
- **前端展示脱敏**：
  - 手机号展示为 `138****1234`
  - 邮箱展示为 `adm***@fy-squad.cn`
  - 真实姓名、身份证号等敏感信息默认隐藏，需申请临时提权后方可查看。

## 2. 权限管理体系 (RBAC)
采用基于角色的访问控制模型（Role-Based Access Control），支持细粒度权限配置。

### 2.1 角色分级
1. **超级管理员 (Super Admin)**：拥有系统最高权限，可分配角色。
2. **社区运营 (Community Ops)**：负责论坛审核、百科内容管理。
3. **财务/赞助管理 (Finance)**：仅拥有赞助订单查看、对账权限，无修改权。
4. **人事/招募管理 (HR)**：负责“加入板块”的申请审批与入队流转。
5. **运维工程师 (DevOps)**：负责“服务器板块”的监控查看与基础配置。

### 2.2 权限颗粒度
- **菜单级**：是否可见某个后台菜单栏目。
- **页面级**：页面内按钮（新增、删除、导出、审核）的显示控制。
- **数据级**：只能看到自己负责的板块数据（如某个分队的申请人）。

## 3. 操作日志与审计留痕
所有后台写操作（POST/PUT/DELETE）必须记录日志，确保 100% 可追溯。
- **日志字段**：`LogID`, `UserID`, `UserName`, `ActionType` (如 UPDATE_WIKI), `TargetID`, `OldData` (JSON), `NewData` (JSON), `IP`, `UserAgent`, `Timestamp`。
- **合规存储**：审计日志仅允许追加，禁止修改和删除。定期归档至冷存储（如腾讯云 COS 归档）。

## 4. 业务数据管理功能
- **百科审核中心**：待审核词条列表、Diff 差异对比视图（类似 Git Diff）、一键打回/通过。
- **论坛治理中心**：敏感词高亮预警列表、用户封禁管理（禁言 24h/7天/永久）、删帖回收站。
- **赞助财务看板**：订单流水明细、对账单导出、异常掉单手动补发权益工具。
- **招募流转中心**：申请人简历池、一面/二面流转状态机管理。

## 5. 数据可视化看板
后台首页提供核心指标（KPI）大屏：
- **流量指标**：日活 (DAU)、新增注册、服务器在线人数折线图。
- **内容指标**：今日新增贴数、待审核词条数、举报未处理数。
- **财务指标**：本月赞助总额、目标达成率进度条。
