# 肥鸭战队官网 - 核心业务板块产品设计规范

## 1. 百科板块 (Wiki)
**目标**：构建战术小队专业知识库，保证内容的高质量与权威性。

### 1.1 全生命周期管理
- **创建与编辑**：采用 Markdown/富文本双模编辑器。支持自动保存草稿。
- **版本回溯**：每次提交生成一个版本快照（Snapshot），用户和管理员可查看历史版本记录，支持“一键回滚”至任意历史版本。
- **审核机制**：
  - LV1-LV3 用户提交修改需进入“待审核队列”，由管理员或高级贡献者审核后发布。
  - LV4+ 用户可免审核直接发布。
- **分类标签**：采用树状分类（如 阵营 > 载具 > 轮式），支持多 Tag 交叉检索（如 `#美军` `#反坦克`）。

### 1.2 用户交互规范
- **纠错反馈**：词条底部提供“指出错误”按钮，弹出划线批注弹窗。
- **收藏与检索**：接入全文搜索引擎（如 Elasticsearch/Meilisearch），支持错别字容错和高亮显示。

## 2. 论坛板块 (Forum)
**目标**：沉淀社区文化，保障交流秩序。

### 2.1 内容与互动机制
- **板块分类**：战术研讨、赛事招募、日常灌水、新人求助。
- **互动规范**：点赞（Upvote）、嵌套评论（楼中楼结构）、引用转发。
- **防刷屏与风控**：
  - 新注册账号 24 小时内禁止发帖。
  - 同一用户发帖间隔最短 60 秒，评论间隔 15 秒。
  - 接入第三方文本安全审核 API（如腾讯云内容安全），自动拦截涉政、暴恐、违禁词汇。

## 3. 服务器板块 (Server)
**目标**：透明化服务器状态，保障基础设施安全。

### 3.1 监控与展示
- **指标体系**：CPU使用率、内存占用、网络带宽、实时 TPS（Tick per second）、当前玩家队列。
- **前端展示**：使用 WebSocket 实时推送服务器状态，避免高频轮询拖垮接口。

### 3.2 架构与安全策略
- **负载均衡**：使用 Nginx/云原生 LB 分发 Web 请求。
- **安全防护**：
  - 强制开启 WAF（Web 应用防火墙），防御 SQL 注入、XSS 攻击。
  - 游戏端口与 Web 端口物理/逻辑隔离，防止针对游戏服的 DDoS 波及官网。
- **灾备方案**：数据库每天凌晨自动全量备份，每 4 小时增量备份，异地冗余存储。

## 4. 赞助板块 (Sponsor)
**目标**：合规、透明、自动化的赞助流水线。

### 4.1 赞助流程
- **档位设置**：配置化管理，如“初级赞助 30元/月”、“高级赞助 100元/月”。
- **支付对接**：对接微信支付/支付宝原生 Native 支付，采用异步回调（Webhook）确认订单，防止前端篡改支付结果。
- **权益发放**：支付成功回调后，系统自动通过 RCON 接口向游戏服务器下发白名单/VIP 权限，并点亮官网专属 Badge。
- **透明公示**：自动生成“月度赞助感谢名单”滚动墙（默认隐藏具体金额或做模糊处理，保护隐私）。

## 5. 加入板块 (Join)
**目标**：标准化人才招募漏斗。

### 5.1 申请与流转
- **表单校验**：Steam ID（校验 64 位数字格式）、游戏时长（要求提供截图或绑定 Steam API 自动读取）。
- **状态机逻辑**：`待处理` -> `一面邀约` -> `考核中` -> `通过/婉拒`。
- **入队引导**：审核通过后，自动发放欢迎站内信，包含新手群号、YY/KOOK 频道密码及新人必读 Wiki 链接。
