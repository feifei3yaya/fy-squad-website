# FY 战术黄主题色卡规范

## 1. 设计目标

- 唯一主色：战术级琥珀黄，承担主按钮、主标题、核心焦点、关键确认。
- 基底色：炭黑、深灰、哑光黑，确保军事氛围稳定克制。
- 点缀色：金属灰、哑光银，仅用于结构分层和信息承载。
- 功能色：队友信息使用冷蓝，敌情使用警示红，中立/负载使用战术橙。
- 可访问性：正文与关键标签优先采用 `fy-silver` / `fy-steel` 作为文本色，避免直接在深底上使用低对比黄。

## 3. 使用分层

### 3.1 主色层

- `fy-amber`：仅用于需要第一视觉优先级的元素。
- 适用场景：主按钮、主导航激活、核心进度、一级标题强调、关键状态确认。

### 3.2 结构层

- `fy-dark / fy-panel / fy-surface / fy-edge` 共同组成 UI 骨架。
- `fy-dark` 负责场景压暗，`fy-panel` 负责主面板承载，`fy-surface` 负责悬浮与输入态，`fy-edge` 负责军事式描边。

### 3.3 文本层

- 标题与大数值：`fy-silver`
- 正文与说明：`fy-steel`
- 弱化辅助信息：`fy-steel` 搭配透明度

### 3.4 功能层

- 队友/链路/可联动：`fy-info`
- 风险/敌情/错误：`fy-red`
- 中立/负载/提醒：`fy-orange`

## 4. 关键界面落地策略

### 4.1 主菜单

- 主按钮、主标题、顶部主状态使用 `fy-amber`
- 导航与说明文字使用 `fy-silver` / `fy-steel`
- 面板和菜单卡片以 `fy-panel + fy-edge` 构成层级

### 4.2 对战 HUD

- 夺点进度、主罗盘焦点使用 `fy-amber`
- 友军、语音链路与稳定状态使用 `fy-info`
- 敌情热点、伤亡与封锁告警使用 `fy-red`
- 高负载、补给压力、争夺中状态使用 `fy-orange`

### 4.3 队内指挥部面板

- 主命令、目标位置、关键部署路径使用 `fy-amber`
- 友军态势、通信模块使用 `fy-info`
- 地图威胁标识使用 `fy-red`
- 地图辅助层、表格、分隔线使用 `fy-edge + fy-steel`

## 5. 新旧配色切换映射

| 旧令牌/旧逻辑 | 新方案 |
| --- | --- |
| 军绿色主色 | 替换为 `fy-amber` 战术黄主色 |
| 绿色描边/弱边界 | 替换为 `fy-edge` 黄褐描边 |
| 绿色在线状态 | 替换为 `fy-info` 队友蓝 |
| 亮黄高负载 | 收敛为 `fy-orange` 中立负载橙 |
| 杂色分类（青、紫、亮红） | 收束为 `fy-amber / fy-info / fy-orange / fy-silver / fy-red / fy-steel` 六类体系 |

## 6. 无缝替换说明

- 主题层以 `tailwind.config.js` 和 `src/index.css` 为核心入口。
- 大量旧的 `fy-green-dim` 结构类保留原类名，但视觉值已改为战术黄褐边界，减少全量类名改动风险。
- 页面级显式状态色已改为语义色：`fy-info / fy-orange / fy-red`，避免继续继承旧绿色视觉。
- Wiki 分类、赞助等级、消息和活动状态已同步清理，避免旧绿与新黄并存。
