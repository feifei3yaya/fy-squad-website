# 项目收口清单

本文档用于整理"肥鸭战队社区官网"从当前展示型前端，推进到可持续开发、可部署、可扩展项目的收口任务。

## 使用方式

- `P0`：必须优先完成，直接影响开发、预览或上线
- `P1`：建议在进入业务开发前后尽快完成
- `P2`：优化项，可在主要功能闭环后逐步推进

---

## P0 基础收口

### 1. 文档与工程说明

- [ ] 补齐根目录 `README.md`，明确项目定位、技术栈、目录结构、启动方式、部署方式
- [ ] 增加"新成员上手说明"，明确 Node.js 版本、依赖安装方式、常用命令
- [ ] 明确 `encoding-skill/` 与主项目的边界，避免误认为业务代码

### 2. 路由与部署可用性

- [ ] 确认当前 `BrowserRouter` 的部署策略
- [ ] 为静态部署环境配置路由回退规则
- [ ] 验证所有一级路由可直接访问
- [ ] 验证刷新二级路由不会出现 404

### 3. 资源与路径统一

- [ ] 明确 `public/` 与 `src/assets/` 的使用边界
- [ ] 统一图片、视频、logo 的引用规范
- [ ] 检查是否存在重复资源、失效资源或历史残留素材
- [ ] 清理无实际引用的图片、视频和备份资源

### 4. 项目可运行性检查

- [ ] 确认 `npm install`、`npm run dev`、`npm run build`、`npm run preview` 全流程可用
- [ ] 运行 `npm run check`
- [ ] 运行 `npm run lint`
- [ ] 处理当前影响构建、预览或交付的阻塞问题

---

## P1 业务收口

### 5. 数据层整理

- [ ] 识别所有硬编码演示数据
- [ ] 将页面内静态数据抽离为统一的数据配置层
- [ ] 区分"演示数据""待接接口数据""真实配置数据"

### 6. 页面交互闭环

- [ ] 梳理登录页的真实需求与接口契约
- [ ] 梳理加入申请页的提交流程与字段校验
- [ ] 梳理联系页表单提交流程
- [ ] 梳理论坛、消息、个人中心是否需要真实后端支持

### 7. 状态与错误处理

- [ ] 设计登录态、用户信息、消息等状态管理方案
- [ ] 为表单增加加载态、成功态、失败态
- [ ] 增加错误提示和异常兜底
- [ ] 为空数据场景补齐空状态展示

### 8. 页面一致性

- [ ] 检查各页面视觉风格是否统一
- [ ] 检查按钮、标题、间距、卡片、导航的交互一致性
- [ ] 校正文案风格，统一中文术语与英文标签

---

## P2 优化收口

### 9. 性能优化

- [ ] 压缩大图和视频资源
- [ ] 为高体积资源增加懒加载策略
- [ ] 评估首页视频背景对首屏性能的影响
- [ ] 检查构建产物体积并进行按路由或按模块拆包

### 10. 代码清理

- [ ] 清理未使用组件
- [ ] 清理未使用 hooks
- [ ] 清理未使用依赖
- [ ] 清理调试代码、无效注释和历史残留逻辑

### 11. 可维护性建设

- [ ] 增加环境变量说明
- [ ] 增加接口配置说明
- [ ] 补充组件复用规范
- [ ] 视需要补充基础测试策略

### 12. 上线质量保障

- [ ] 增加 404、错误边界、加载中等兜底体验
- [ ] 补齐 favicon、页面标题、基础 SEO 信息
- [ ] 验证移动端适配效果
- [ ] 验证主流浏览器兼容性

---

## 建议执行顺序

1. 先做 `P0`，确保项目结构、资源、构建、部署路径清晰可控
2. 再做 `P1`，把当前展示页面推进为具备真实业务闭环的站点
3. 最后做 `P2`，提高性能、维护性与上线质量

## 建议第一批落地任务

如果马上开始实做，建议优先做下面 5 项：

1. 统一 `public/` 与 `src/assets/` 的资源规范
2. 修正文档并补部署说明
3. 跑通 `lint`、`check`、`build`
4. 梳理登录 / 加入申请 / 联系表单的数据流
5. 清理未使用依赖与未接入代码
